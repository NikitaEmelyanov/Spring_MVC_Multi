________________1, Автоматическая генерация значения ID (Автоинкремент, Serial)_______________________
Как правило это создается на стороне БД, а не на стороне Spring приложения. В PostgreSQL
автоматической генерации id можно добиться с помощью механизма sequence. Для этого в консоле БД
нужно создать этот самый sequence (последовательность). Sequence это такой объект в БД, который
выдает несколько целых чисел по порядку

 create sequence first_sequance
 select nextval('first_sequance') - данный метод возвращает число, и при каждом его последующем
 вызове это число инкриминируется
 drop sequence  first_sequance - удаление sequance

 ________________Создание таблицы Person c автоматической инкрементацией id_________________________

 create table Person (  -  Создание новой таблицы Person
     id SERIAL,         - SERIAL - это псевдо тип данных, который помимо создания целого числа
     name varchar,        создает еще и последовательность sequance, и при добавлении новой строки
     age int,             каждый раз будет срабатывать select nextval('название sequance ')
     email varchar      - varchar, аналог String в SQL
 )

 insert into Person (name, age, email) values ('Tim',30,'test@mail.ru')
 Далее можно добавлять в таблицу значения, кроме id, так как id генерируется автоматически
insert into Person (name, age, email) values ('Tim123',30,'tin123@mail.ru')
insert into Person (name, age, email) values ('Bob',35,'bob@mail.ru')
Для проверки выполняем 3 раза с разными значениями
select * from person - проверяем все ли корректно в таблице
