___________________________________Аннотация @ModelAttribute________________________________________
1) Может аннотировать метод
@ModelAttribute("headerMessage")
public String populateHeaderMessage(){
    return "Welcome to our website!";
}
2) Может аннотировать аргумент метода
@PostMapping()
public String create(@ModelAttribute("person") Person person){
    personDAO.save(person);
    return "redirect:/people";
}
_____________________________________Аннотация метода________________________________________________
1) В модель в каждом методе текущего контроллера добавляет ключ-значение
2) Используется для добавления тех пар ключ-значение, которые нужны во всех моделях этого контроллера
3) Любая модель из этого контроллера по умолчанию будут иметь значение с ключом headerMessage
4) Может добавлять в модель любой объект

________________________________Аннотация аргумента метода___________________________________________
1) В модель будет положен новый объект со значениями полей по умолчанию (0, null и так далее)

_______________________Деталь реализации PATCH, DELETE и других запросов_____________________________
|                HTTP методы:                   |                         HTML 5:                   |
|                GET                            |                         GET                       |
|                POST                           |                         POST                      |
|                PATCH                          |                                                   |
|                DELETE                         |                                                   |
|                PET                            |                                                   |
|                HEAD                           |                                                   |
|                CONNECT                        |                                                   |
|                OPTIONS                        |                                                   |
|                ...                            |                                                   |
-----------------------------------------------------------------------------------------------------
Поскольку HTML 5 может принимать только 2 метода GET и POST, остальные методы, которые могут
потребоваться при работе (PATCH, DELETE, PUT) запросы передаются с помощью POST запроса, но в
скрытом поле (тип "hidden") со специальным названием _method указывается желаемый HTTP метод.

<form method="post" action="/people/1">
   <input type="hidden" name="_method" value="patch"> = $0
   <label for="name">Enter name: </label>
   <input type="text" id="name" name="name" value="Tom">
   <br>
   <input type="submit" value="Update!">
   </form>

__________На стороне Spring приложения чтение скрытых полей реализуется с помощью фильтра___________

Фильтр - объект, который перехватывает все входящие HTTP запросы. В данном случае фильтр
используется для того, чтобы смотреть на значение поля _method в поступающих HTTP запросах
(Если это поле есть)
